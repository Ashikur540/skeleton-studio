// WHAT'S WORKING

Composition

Three-pane operate layout (input → preview → inspector) fits the tool's work pattern perfectly. Each pane scrolls independently.

State Coverage

Empty states, error banners, disabled controls, loading states, and success feedback are all handled. StarterBrowser handles empty search results.

Interaction Polish

Pointer-capture drag with undo snapshot batching, Figma-style scrub fields with shift-multiplier, and deterministic skeleton variance on repeat nodes.

UI Kit Consistency

shadcn/ui primitives are customized with a unified language: rounded-4xl buttons, glass Select popovers, line-variant tabs with underline indicator.

Export Experience

Split-pane export modal with React/HTML tabs, read-only CodeMirror, summary stats, Copy feedback with 2-second timeout, and Download action.

Keyboard Shortcuts

⌘Z/⌘⇧Z undo, Delete to hide, arrow keys to nudge. Smart skip logic avoids firing inside CodeMirror, inputs, dialogs, and popovers.

// PRIORITY ISSUES
P0 · Critical

No Responsive Behavior — Desktop-Only

The three-pane layout uses fixed widths (left: 420px, right: 288px) with zero breakpoints. At 768px the center canvas collapses to ~40px. At 375px (iPhone) the tool is completely unusable with horizontal overflow. The Export modal has responsive max-widths, but this is the only responsive behavior in the app.

FIX: Stack panes vertically below 900px with tab navigation between Input / Preview / Properties. Use /design responsive to recompose.
✓ RESOLVED

prefers-reduced-motion — Added

Added `@media (prefers-reduced-motion: reduce)` block to globals.css that freezes the shimmer keyframe and collapses all animation/transition durations to 0.01ms. The shimmer and pulse animations are now disabled for users who set this OS-level preference.

P1 · High

Canvas Blocks Not Keyboard-Accessible

Skeleton block selection and inspection requires clicking. There is no tabIndex on rendered skeleton nodes, no onKeyDown handler for Enter to select, and no focus-visible styling. Keyboard-only users cannot interact with the preview canvas at all.

FIX: Add tabIndex={0} + onKeyDown to skeleton-renderer.tsx blocks, plus focus-visible ring styling matching the selection border. Use /design interaction for keyboard canvas support.
P1 · High

Low Contrast on Muted Text (Dark Mode)

--muted-foreground at oklch(0.723 0.014 214.4) on --background at oklch(0.148 0.004 228.8) yields approximately 3.9:1 contrast — below the WCAG AA 4.5:1 threshold for normal text. Affects section headers, descriptions, sub-labels, and footer hints throughout the interface.

FIX: Increase --muted-foreground lightness to L ≥ 0.78 in dark mode. Run /design typeset for systematic contrast review.
✓ RESOLVED

text-[10px] to text-[11px] — Bumped in 7 files

All ten instances of text-[10px] across 7 files (page header badge, resize overlay badge, padding box, shortcuts modal, starter browser, section wrapper, layers tab) were bumped to text-[11px]. One more pixel helps legibility without disrupting the compact tool aesthetic.

The permanently-disabled Switch in the Export modal's sidebar has been replaced with a plain text note confirming all output uses Tailwind v4-compatible syntax. The unused Switch and Label imports were cleaned up.

P3 · Low

Primary Hue Lacks Domain Connection

The chartreuse primary (oklch(0.768 0.233 130.85)) is vibrant and intentional, but has no clear rationale tied to the skeleton/placeholder/scaffolding domain concept. It could belong to any green-branded dev tool without feeling specific to this product.

SUGGEST: Run /design recolor with a domain-aligned emotional arc. Consider cooler, more structural tones (slate-blue, neutral-cool) that evoke scaffolding rather than vibrancy.
